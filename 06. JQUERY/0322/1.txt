
* 블록체인 = DB
  INSERT만 있음. UPDATE와 DELETE는 없음. - 데이터 조작이 불가능함.
  이미 입력된 데이터를 가져가거나 특정 데이터만 빼갈수 없음 - 체인으로 다 연결이 된 상태라서.

* J QUERY에서 쓰는것들
  focus (초점 맞추기),      val (값을 가져옴),      trim (공백 제거)
  children (부모 기준으로 자식 요소를 알고 싶을때),      find (부모 기준으로 자식의 자식 요소를 알고 싶을때)
  on (클릭, 바인딩)
  append (추가 - 포스트잇처럼)
  show (보여줌),      hide (숨김)
  메소드 체이닝 (리턴 값이 있는 메소드끼리 연결)
  정규 표현식


-----------------------------------------------------------------------------------------------


* J QUERY 장단점
  1. 장점
     쉽고 빠르게 Server와 통신하는 아키텍쳐를 구성.
     레퍼런스(참고 문서)가 많다.
  2. 단점
     jQeury로 아무리 클린하게 코드를 구현해도 
     웹 사이트가 점점 커지면 코드 복잡도가 올라가고 유지보수가 힘듦.

  1. for-each = 범위 지정을 하지 않음. 
       array = [1,3,4,5,10]
       for(i:array)   // 배열 안의 값을 다 돌면 끝
  2. for(i=0; i<3; ++i)
