

* 애플리케이션 요구사항 분석 (과목명)

  - 애플리케이션(소프트웨어)을 만들어 가는 과정
       1. 요구분석 ( 기능 명세서, 권한 정책 , ... )
       2. 설계 ( ERD ( DB 테이블 명세서 ), UI 스토리 보드, 클래스 명세서 , ... )
       3. 구현 ( 퍼블리싱, API 개발 ( 백엔드 개발 ), 테이블 생성 , ...  )
       4. 테스트 ( optional ) - 개발 기간에 따라 선택 가능
                              - "Postman, JUnit, Jest, ..." 의 도구(소프트웨어)를 이용해서 테스트
       5. 유지보수 (  )

  - 애자일 방법론 ( 구글에서 창시 ) - 스타트업에서 많이 사용 함
                                  - 많은 방법론 중에서 스크럼을 많이 사용  ex) 뱅크샐러드, toss, 애플 등등
    애자일 : 민첩한, 날렵한
    스크럼 : 애자일 방법론 과정의 하나
       특징 1. 솔루션에 포함할 기능 및 개선점에 대한 우선 순위를 부여 함.
       특징 2. *** 개발 주기( 요구분석 ~ 구현까지의 과정 )가 30일을 넘으면 안됨
                - 고객한테 보여주기 위해서 빠르게 기능만 추가 했을때의 주기임
                - 개발 주기마다 실제 동작 할 수 있는 결과를 제공 해야 됨
       특징 3. 개발 주기( 한달 )마다 적용 할 기능과 개선 목록을 제공해야 됨
       특징 4. 날마다 15분정도 회의를 해야 됨 - 공적인 얘기들로.
       특징 5. 항상 팀 단위로 생각 해야 됨. - 구분 없는 열린 공간을 유지 해야 됨
       특징 6. *** 기능 개발은 3일 안쪽으로.
       특징 7. 제품 백로그( back log ) : 개발 할 제품에 대한 요구사항 목록
                - 제품 백로그( Product Backlog ) 예시 = 요구 사항 분석
                     프로젝트 명 - 학생 전용 깃허브 플랫폼 ( 국가 프로젝트로 가정 )
                        백로그 작성 예시
                            1) 학생은 작성한 소스(코딩)를 업로드 해야 함
                            2) 학생은 로그인 후에 본인 소스를 볼 수 있어야 함
                            3) 학생은 업로드 한 소스를 다운로드 받을 수 있다
                            4) 선생님은 학생들 소스를 관리 및 확인 할 수 있다
                            5) 선생님은 업로드 한 소스를 다운로드 받을 수 있다
                            스프린트( 04/12 ~ 05/06 ) - 목표 : 큰 기능을 우선 구현
                            기능명세서 - a) 소스 업로드 구현
                                        b) 소스 확인 구현
                                        c) 로그인 구현
                                        d) 소스 다운로드 구현
                                        e) 소스 관리 구현
       결론 - 개발 주기( 스프린트 )가 모여 하나의 어플리케이션이 탄생 함.
               스프린트 : 짧은 거리( 한달 )를 전력 질주.

* 백로그 문제

  프로젝트 명 : 사내 메신저 플랫폼
  백로그 작성 예시
    1. 사용자는 로그인 후 메일 업무를 하기 위한 선택을 하기를 원함
    2. 사용자는 메일함의 목록을 볼 수 있어야 함
    3. 사용자는 받은 최신 메일 목록을 보기를 원함
    4. 사용자는 선택 한 메일을 작성 하기를 원함
    5. 사용자는 원하는 메일을 삭제 하기를 원함
  기능 명세서 작성 후에 작업 우선순위를 결정
    1. 로그인 기능
    2. 메일 선택 기능
    3. 메일 목록 열람 기능
    4. 최신 메일 목록 열람 기능
    5. 선택 한 메일 작성 기능
    6. 특정 메일 삭제 기능

* 컬렉션즈

  <> = 제네릭
     ArrayList, *** Hash Map
        Hash = Key와 Value로 이루어진 자료구조 ( JSON과 같음 )
               Key 중복 X, Value 중복 O

  HashMap Value에 Object를 하는 이유
     Value에 어떤 값(데이터타입)이 올지 예상 불가능. 그래서 Object로 데이터타입을 모두 받는다.

  모든 객체지향 프로그램에서, 모든 클래스는 디폴트로 Object를 갖고 있다.
