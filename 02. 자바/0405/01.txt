

* 상속
  클래스(객체)에 대한 개념이 부족하면 상속이 힘들다


----------------------------------------------------------------------


* 자바
  프로젝트 -> 패키지 -> 클래스
    1. 프로젝트 생성
       1) 프로젝트 이름은 대문자 or 소문자
          ex)  main_file, mainFile
    2. 패키지 생성 (생략 가능)
       1) 패키지 이름은 소문자
          ex) mainFile
    3. 클래스 파일 생성
       1) 클래스 이름 첫글자는 대문자


----------------------------------------------------------------------


* 지금 쓰는 자바 버전 = 12


----------------------------------------------------------------------


* new를 이용해서 클래스 호출을 할 때, 생성자가 자동으로 호출 됨


----------------------------------------------------------------------


* 생성자 특징
  1. 생성자 이름은 클래스 이름과 동일해야 됨
  2. return type이 없다.


----------------------------------------------------------------------


* 오버로딩
  1. 메소드 이름은 동일
  2. 파라미터(매개변수, 인자)는 다르게 해야 됨


----------------------------------------------------------------------


* 데이터 타입
  1. 정적 : 배열, float, int, double, char
  2. 동적 : 배열, String, Class


* 자바에서는 정적과 동적을 따로 관리 함 - 서로 저장 하는 공간이 다름
  1. 정적 관리 : 
  2. 동적 관리 : Heap(메모리) Area


----------------------------------------------------------------------


* 클래스를 만드는 이유 - 클래스마다 기능이 다름


* 상속
  1. 클래스마다 중복으로 들어가는 메소드는, "상속(공통 기능)" 클래스를 만든 후에 가져와서 사용
  2. 공통 기능 클래스가 "부모 클래스 or 슈퍼 클래스"
        메소드, 필드변수를 제공하는 주체
        부모 클래스는 자식의 기능(메소드, 필드변수)를 사용 할 수 없음
     공통 기능 클래스에서 가져와서 쓰는 클래스가 "자식 클래스 or 서브 클래스"
        자식 클래스는 부모의 기능(메소드, 필드변수)를 사용 할 수 있음
  3. 모든 클래스의 부모는 object
  4. 자바는 C++과 다르게 다중 상속이 없다. - 자바는 부모가 한명


----------------------------------------------------------------------


* 문법

  class Pizza extends Object {   // extends = 상속
			   // Pizza가 Object를 상속한다.
			   // 왼쪽이 자식, 오른쪽이 부모
			   // extends Object는 생략 가능

  }
