------------------------------------------------------------------------

* 사람이 작성한 코드를 컴퓨터가 알아듣게 번역 해야 함.
  번역하는 방법 두가지 - 컴파일, 인터프리터

------------------------------------------------------------------------

* 컴파일 - C, C++, JAVA, ...
  1) 다 실행시킨 뒤에 결과값이 나옴 ( 전체 소스코드를 보고 실행 )
  2) 인터프리터와 비교했을 때 실행속도가 빠름
  3) 코드를 수정하면 처음부터 다시 실행 됨.

------------------------------------------------------------------------

* 인터프리터 - JavaScript, Python, ...
  1) 한줄 한줄씩 바로 결과값이 나옴 ( 한줄씩 실행 )
  2) 컴파일과 비교했을 때 실행속도가 느림
  3) 코드를 수정하면 즉시 실행 됨.

------------------------------------------------------------------------

* 오픈 소스 - Mysql, Java, http, 이클립스, ...

  5) 변수 이름은 중복 불가

* 스텝 1
  변수는 재활용이 가능함
	int 돼지국밥 = 7000;
	int 콩나물국밥 = 5000;
	돼지국밥 = 돼지국밥+콩나물국밥
	// 돼지국밥은 12000원이 나옴

* 스텝 2
  컴퓨터가 해석하는 방향은 오른쪽에서 왼쪽으로, 위에서 아래로 내려감.
  나중에 쓴 것에서 정의를 내리면, 그 전에 쓴 데이터는 무시 됨.
	int 돼지국밥 = 7000;
	int 콩나물국밥 = 5000;
	돼지국밥 = 돼지국밥+콩나물국밥
	// 돼지국밥은 12000원이 나옴
	돼지국밥 = 1000;
	// 돼지국밥은 1000원이 나옴

*** 실행 연산자 ( 실무에서 많이 쓰임 )
	int 손흥민 = (10>4) ? 10:30;              // 물음표를 기준으로, 왼쪽에는 '()'를 적고, 오른쪽에는 ':'를 적음
					//  '()'안에는 조건식이 들어감. 조건식이 참이면 ':' 기준으로 왼쪽 값이 나오고, 조건식이 거짓이면 ':' 기준으로 오른쪽 값이 나옴.
	System.out.println(손흥민);


	ex)  변수 = (회원이 로그인 했을때 연산자) ? home : login		// 로그인 성공하면 홈 화면으로, 로그인 못하면 로그인 화면으로
